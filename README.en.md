# Radar Combustível — Near Real-Time Data Pipeline (MongoDB → Redis)

This project implements a near real-time data pipeline using MongoDB as an event source
and Redis as the serving layer, applied to the Radar Combustível platform use case.

📘 Academic Context  
This project was developed as a group assignment for the **In-Memory Database** course
of the **MBA in Technology – FIAP**.

🎯 **Project Objective**  
To design and implement, as a group, a near real-time data pipeline using MongoDB as an event source
and Redis as the serving layer, applied to the Radar Combustível platform use case.
<br><br>
📦 **Scope and Deliverables**  
The solution includes the following components:
- Modeling of the Radar Combustível use case, with coherent entities, events, and queries;
- Implementation of the MongoDB → Redis pipeline, oriented to event-driven updates;
- Appropriate use of Redis data structures (Hashes, Sorted Sets, Geo, Time Series);
- A visualization layer for querying and analysing metrics served from Redis;
- Architecture documentation, technical decisions, and execution instructions.
